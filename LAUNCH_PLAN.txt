 ASCENT TERMINAL — PHASED LAUNCH PLAN
══════════════════════════════════════════════════

This plan takes you from zero to a live, monitored
algo-trading setup in four phases over ~2 weeks.

──────────────────────────────────────────────────
PHASE 1 — Setup & paper trade  (Days 1-3)
──────────────────────────────────────────────────
Goal: environment working, bot running in paper mode.

□ Clone repo on Windows PC
□ Run organise.bat
□ Fill in platform/.env  (API keys + Telegram)
□ pip install -r platform/requirements.txt
□ Set PAPER_TRADE=true
□ Start scalper_bot.py — confirm Telegram messages
□ Start mexc_trend_bot.py in a second terminal
□ Watch for 24 h — check logs, no errors

Success criteria
  • Both bots send Telegram messages
  • No Python exceptions in logs
  • Simulated P&L reported correctly

──────────────────────────────────────────────────
PHASE 2 — Research & tuning  (Days 4-7)
──────────────────────────────────────────────────
Goal: find the best parameters before risking money.

□ Run edge_lab.py on BTC/USDT, ETH/USDT, SOL/USDT
□ Open the output CSV — identify top 3 param combos
□ Update .env with best RSI_OVERSOLD / RSI_OVERBOUGHT
□ Run swing_backtest.py on the chosen params
□ Review the backtest equity curve and drawdown
□ Decide: is the edge real? (Sharpe > 1.0, DD < 20%)

Success criteria
  • At least one param set with Sharpe > 1.0
  • Max drawdown < 20% in backtest
  • Win rate > 45%

──────────────────────────────────────────────────
PHASE 3 — Live micro-test  (Days 8-11)
──────────────────────────────────────────────────
Goal: confirm live execution works with tiny size.

□ Set PAPER_TRADE=false
□ Set POSITION_SIZE_USDT=10  (smallest practical size)
□ Set LEVERAGE=3             (conservative)
□ Run scalper for 3 days
□ Compare live fills vs paper signals
□ Check slippage — is it within 0.05% of signal price?
□ Confirm stop-losses trigger correctly

Success criteria
  • Fills within acceptable slippage
  • Stop-losses fire correctly
  • No runaway positions
  • P&L roughly matches paper expectation

──────────────────────────────────────────────────
PHASE 4 — Scale up  (Day 12+)
──────────────────────────────────────────────────
Goal: increase size gradually as confidence grows.

□ Increase POSITION_SIZE_USDT to 25, then 50
□ Add mexc_trend_bot.py to live rotation
□ Run both bots — monitor for conflicts / over-trading
□ Set a monthly loss limit: if down > 15%, stop & review
□ Review trades CSV weekly — is edge holding?

Scale-up schedule (suggested)
  Week 1:  $10/trade  — prove execution
  Week 2:  $25/trade  — build confidence
  Week 3:  $50/trade  — normal operating size
  Month 2: $100/trade — only if Week 3 was profitable

──────────────────────────────────────────────────
RISK RULES (non-negotiable)
──────────────────────────────────────────────────
  • Never risk more than 2% of account per trade
  • Never run leverage > 10x
  • Always keep PAPER_TRADE=true until Phase 3
  • Keep API key withdrawal permission OFF
  • Review logs daily for the first month
