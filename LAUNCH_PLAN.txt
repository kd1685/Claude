This document is the owner's launch roadmap.  It is NOT a technical
deploy guide — for that see GO_LIVE_STEPS.md.

────────────────────────────────────────────────────────────────────
PHASE 0  ·  OWNER SELF-HOSTING  (now → week 1)
────────────────────────────────────────────────────────────────────

Goal: terminal running on your own VPS, accessible via HTTPS domain.

□ Rent a VPS (Hetzner CX22 / DigitalOcean Droplet / any Ubuntu 22+)
□ Follow GO_LIVE_STEPS.md end-to-end
□ Confirm all smoke-tests pass
□ Point a domain / subdomain at the server IP
□ Set up systemd so it survives reboots
□ Test TradingView webhook from a real alert
□ Run one paper bot for 48 h, review its event log

Success criteria:
  • Terminal loads at https://yourdomain.com with your access key
  • CHART, SIGNALS, AI, ALERTS, BOTS tabs all functional
  • At least one paper order executed from a TV alert

────────────────────────────────────────────────────────────────────
PHASE 1  ·  LIVE TRADING VALIDATION  (week 1–4)
────────────────────────────────────────────────────────────────────

Goal: run the EMA30 trend bot on a small live account;
validate execution end-to-end.

□ Fund MEXC futures sub-account with $200–500 test capital
□ Set EXECUTE_KEY + exchange API keys (IP-whitelisted, no withdrawal)
□ Switch bot to LIVE mode with 0.5–1% risk per trade
□ Monitor daily: check bot events, P&L, slippage vs paper
□ Test the TP/SL watcher fires correctly on a small position
□ Review any "server down" gaps — harden with a health-check cron

Success criteria:
  • ≥ 10 live trades executed with correct size and direction
  • TP or SL watcher fires at least once correctly
  • Draw-down stays within expected range vs backtest

────────────────────────────────────────────────────────────────────
PHASE 2  ·  PERSISTENCE + MULTI-SESSION  (week 4–8)
────────────────────────────────────────────────────────────────────

Goal: no data loss on server restart.

□ Add SQLite (via SQLAlchemy or aiosqlite) to persist:
    alerts, execution log, bot state, exit plans
□ Migrate in-memory stores to DB-backed stores in main.py
□ Verify alerts + bot positions survive uvicorn restart
□ Add a simple admin endpoint to export trade history as CSV

────────────────────────────────────────────────────────────────────
PHASE 3  ·  EXCHANGE-NATIVE TP/SL  (week 8–12)
────────────────────────────────────────────────────────────────────

Goal: replace server-polled TP/SL watcher with native orders.

□ Use ccxt create_order with stopLoss / takeProfit params
    (supported on Binance, MEXC, Bybit, OKX)
□ Keep server watcher as fallback for exchanges that don't support it
□ Update UI to show whether plan is "exchange-native" or "server-watched"

────────────────────────────────────────────────────────────────────
PHASE 4  ·  OPTIONAL MONETISATION  (if desired)
────────────────────────────────────────────────────────────────────

Option A — Invite-only access keys
  • Generate N keys; hand out to friends / community
  • Each key has a trade-count or time expiry stored server-side
  • No payment infra needed — manual distribution

Option B — Stripe subscription
  • Add /api/subscribe endpoint with Stripe Checkout
  • On payment, generate + email an access key
  • Store keys + expiry in SQLite

Option C — Keep it private
  • Use only for your own trading — no distribution
  • Lowest maintenance; highest alpha retention

────────────────────────────────────────────────────────────────────
RISKS & MITIGATIONS
────────────────────────────────────────────────────────────────────

Risk: Server downtime during open position
  → TP/SL watcher won't fire.  Mitigation: set exchange-native stops
    as backup (Phase 3), monitor with UptimeRobot free tier.

Risk: API key compromise
  → Whitelist server IP on exchange.  No withdrawal permission.
    Rotate keys if suspicious activity.

Risk: Claude API cost spike
  → Responses are cached 15 min per asset.  Set a monthly spend
    cap in Anthropic console.

Risk: Bad fill / runaway bot
  → USER_CAP_USD caps every order.  Bot max-positions param limits
    concurrent exposure.  Kill switch: set EXECUTE_KEY to garbage.
